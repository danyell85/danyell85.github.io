Arrays and Hashes

Arrays and hashes in ruby are both used to hold collections of objects. Depending on how these collections should be accessed will determined if an array or hash is the best for the job.

Arrays

Arrays are ordered collections of any objects. Objects can be strings, integers, floats or other arrays. The array orders itself in an integer-index. Integers start at 0 so to find the first object in an array the user can input array[0].
Example: array = [obj1, obj2, obj 3, ...objx]
In the above exmaple obj1 has an index of 0, obj2 has an index of 1 and so on.
Arrays are handy for storing data that is ordered.

Hashes

Hashes are similar to arrays. They are a collection of ojects and there values. Unlike arrays, hashes arent index by integers but assigned values. It is like a dictionary. When looking up a word(key in hash) there is a definition next to the word(value for hash).To search for something in a hash the user can input hash["key1"] and value1 will be returned..
Example : hash = {key1 => value1, key2 => value2, key3 => value3, keyx => valuex}
